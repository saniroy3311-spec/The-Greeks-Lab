# A1 Standalone System — Trading Agent Playbook & Guide
This guidebook details the architecture, strategy logic, chart visualizations, and step-by-step execution guidelines for the AI Trading Agent operating the **A1 Standalone System** powered by Kronos AI and CandL Charts.

---

## 1. System Overview & Philosophy
The A1 Standalone System is designed as a **confluence-based execution filter**. It does not auto-trade; instead, it intercepts entry triggers generated by a classic moving average crossover strategy and subjects them to rigorous validation gates using deep-learning time-series forecasting (Kronos AI) and non-parametric historical pattern matching (Echoes Engine).

### The Three Pillars:
1. **System 1: EMA Crossover (Momentum Trigger)**
   Identifies immediate trend momentum using the 5 EMA and 10 EMA.
2. **System 2: Kronos AI (Deep Learning Forecast)**
   Analyzes the last 400 candles and predicts the next 24 closes. Confirms direction, expected move size, and prediction confidence.
3. **System 3: Echoes Similarity Engine (Historical Analogy)**
   Performs a Z-normalized Euclidean distance scan over historical data to identify the top 5 closest matches. Evaluates subsequent win rate and reward/risk asymmetry.

- **Flask Backend (`server.py`)**: Mounts WebSocket connections to Delta Exchange (real-time crypto) and polls yfinance (NSE indices). Runs prediction inference using PyTorch.

- **CandL Charts Canvas Core (`candl-charts.js`)**: Framework-free high-performance canvas engine rendering candlestick series, drawings, and overlays.
- **Oracle & Echoes (Client-Side)**: Uses the compiled `lab` module from CandL Charts inside the client browser to run CPU-intensive block-bootstrap path simulations (Oracle) and Euclidean distance scans (Echoes) instantly.

## 3. The A1 Strategy Core (6 Entry Gates)
Every potential trade trigger must pass **all 6 validation gates** to be approved (`BUY` or `SELL`). If any gate fails, the decision defaults to `SKIP`.

### Gate 1: Trend Alignment
- **Long**: Fast EMA (5) > Slow EMA (10).
- **Short**: Fast EMA (5) < Slow EMA (10).

### Gate 2: Crossover Freshness
- Prevents chasing mature trends.
- The crossover must have occurred **at least 5 bars ago** (`bars_since_cross >= 5`).

### Gate 3: Candle Body Confirmation
- The trigger candle's close must confirm the direction:
  - **Long**: Candle must be Green (`close > open`).
  - **Short**: Candle must be Red (`close < open`).

### Gate 4: Kronos AI Directional Agreement
- Kronos AI prediction direction must match the EMA trend:
  - **Long**: Kronos direction must be `Long`.
  - **Short**: Kronos direction must be `Short`.

### Gate 5: Kronos Strength & Expected Move
- Prediction confidence must be **at least 80%** (`confidence >= 80`).
- Predicted move percentage must be **at least 0.40%** in magnitude:
  - **Long**: `move_pct >= +0.004`.
  - **Short**: `move_pct <= -0.004`.

### Gate 6: Echoes Historical Win Rate & Asymmetry
- Matches historical windows that look like the current 30-candle stroke.
- **Long**: Historical win rate of the matched paths must be `WinRate >= 80%` (at least 4 out of 5 paths went up) and asymmetry `Asymmetry >= 3.0`.
- **Short**: Historical win rate must be `WinRate <= 20%` (at least 4 out of 5 paths went down) and asymmetry `Asymmetry >= 3.0`.

## 4. The 8 Skip Conditions
The trading agent must block execution (set output to `SKIP`) if **any** of the following conditions are met:

1. **ATR Spike**: The current candle range (`high - low`) exceeds 2.5x the 14-period Average True Range (ATR), indicating an unstable breakout.
2. **Extreme Volume**: Current volume is > 4.0x the 20-period moving average of volume.
3. **Flat Range**: The high-to-low range over the last 10 candles is under 0.15% of the current price, indicating zero volatility.
4. **EMA Compression**: The gap between EMA 5 and EMA 10 is less than 0.02% of the close price, indicating a lack of momentum.
5. **Session Clock Block (Indices)**: No trades allowed in the first 15 minutes of index opening or the last 30 minutes before closing.
6. **Low Confidence**: Kronos AI confidence score is below 80%.
7. **Oracle Dispersion**: The P95-to-P5 price envelope width is wider than 2.0% of the price, indicating high future paths uncertainty.
8. **Pattern Contradiction**: Echoes historical matches show contradictory outcomes (e.g. 3 paths up, 2 down).

## 6. Trading Agent Execution Protocol

### Step-by-Step Validation Loop:
1. **Monitor Crossovers**: Watch the chart for an EMA 5 and EMA 10 crossing event.
2. **Fetch API Endpoint**: Once a crossover occurs, query the A1 API endpoint:
   `GET /api/a1/<symbol>/<tf>`
3. **Review JSON Schema**: Ensure the response has a `BUY` or `SELL` decision:
   ```json
   {
     "symbol": "BTC",
     "timeframe": "5m",
     "decision": "BUY",
     "gates": {
       "gate1_trend": true,
       "gate2_fresh": true,
       "gate3_color": true,
       "gate4_direction": true,
       "gate5_strength": true,
       "gate6_echoes": true
     },
     "skip_reasons": []
   }
   ```
4. **Cross-Verify on Visual Dashboard**:
   - Verify that the solid orange forecast line points in the direction of the trade.
   - Ensure the blue Oracle cone does not show massive dispersion (P95-to-P5 range should be tight).
   - Ensure the connection status dot is green (`Live`), indicating you are acting on fresh data.
5. **Execution**: If `decision == "BUY"` and no visual anomalies exist, execute a long entry. If `decision == "SELL"`, execute a short entry. If `decision == "SKIP"`, remain flat.
